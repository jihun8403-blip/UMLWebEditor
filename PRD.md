# UML Web Editor

## Product Requirements Document (PRD)

* * *

# 1. 제품 개요

본 제품은 별도의 설치 없이 웹 브라우저에서 실행 가능한 **정적 웹 기반 UML 다이어그램 편집기**이다.

사용자는 UML 다이어그램을 생성하고 편집하며, UML 요소의 속성을 데이터 기반으로 관리할 수 있다. 생성된 다이어그램은 JSON 형식으로 저장하거나 불러올 수 있으며, 이미지 파일 형태로 내보내기(export) 기능을 제공한다.

본 제품은 단순한 도형 기반 드로잉 도구가 아니라 **UML 모델을 구조화된 데이터로 관리하는 모델 기반 에디터**를 목표로 한다.

초기 버전(MVP)은 UML 작성 및 편집 기능에 집중하며, 향후 확장으로 UML 전체 지원, 코드 생성, 리버스 엔지니어링, XMI 호환 기능 등을 제공할 수 있도록 구조적으로 설계한다.

* * *

# 2. 제품 목표

본 제품의 주요 목표는 다음과 같다.

* 웹 브라우저에서 **설치 없이 UML 다이어그램 작성 가능**
    
* UML 요소를 **데이터 모델 기반으로 관리**
    
* 직관적인 UI로 **빠른 설계 작업 가능**
    
* 다이어그램을 **JSON 형식으로 저장 및 재사용**
    
* 고해상도 이미지 및 PDF로 **설계 문서 공유 가능**
    

* * *

# 3. 제품 범위

초기 버전(MVP)은 단일 사용자 환경을 대상으로 한다.

협업 기능이나 코드 분석 기능은 MVP 범위에서 제외하며 향후 버전에서 고려한다.

지원 범위

* UML 다이어그램 생성 및 편집
    
* UML 요소 속성 관리
    
* JSON 기반 프로젝트 저장 및 불러오기
    
* 이미지 및 PDF export
    
* 로컬 브라우저 기반 실행

# 7. UML 요소 속성 관리

각 UML 요소는 단순한 시각 객체가 아니라 **데이터 기반 객체**로 관리되어야 한다.

## 7.1 클래스 속성

클래스 다이어그램에서 다음 속성을 지원해야 한다.

* 클래스 이름
    
* 속성(Attribute)
    
* 메서드(Method)
    
* 가시성
    

가시성 표기

* Public +
    
* Private -
    
* Protected #
    

* * *

## 7.2 스테레오타입

UML 확장 메커니즘을 지원해야 한다.

예시

<<interface>>  
<<entity>>  
<<control>>

스테레오타입은 시각적으로 구분되어 표시되어야 한다.

* * *

## 7.3 검색 기능

복잡한 다이어그램에서 특정 요소를 찾기 위한 검색 기능을 제공해야 한다.

검색 대상

* 클래스 이름
    
* 속성 이름
    
* 메서드 이름
    
* 요소 ID
    

검색 결과는 캔버스에서 하이라이트되어야 한다.

# 15. 향후 확장 계획 (Future Scope)

다음 기능은 초기 버전에서 제외되며 향후 버전에서 검토한다.

* UML 전체 다이어그램 지원
    
* 스마트 라우팅 (자동 선 경로 최적화)
    
* 코드 생성 (Forward Engineering)
    
* 리버스 엔지니어링 (Code → UML)
    
* XMI Import / Export
    
* Git 기반 버전 관리 연동
    
* 실시간 협업 기능
    

* * *

한 가지 현실적인 조언을 덧붙이자.  
**UML 에디터의 핵심 난이도는 “도형”이 아니라 캔버스 엔진이다.**

특히 이 세 가지가 기술적으로 제일 중요하다.

* **선 연결(anchor 시스템)**
    
* **Undo/Redo 히스토리 모델**
    
* **요소 데이터 모델**
    

이 세 개를 제대로 설계하면 기능은 계속 붙는다.  
반대로 여기서 구조가 틀어지면 기능을 하나 추가할 때마다 시스템이 삐걱거리기 시작한다. 소프트웨어의 뼈대가 관절염 걸린 것처럼 말이다.

원하면 다음 단계로 이어서 **System Design 문서(SDS)**를 만들어 줄 수 있다.  
그 문서는 보통 이걸 포함한다.

* 전체 아키텍처
    
* 캔버스 렌더링 엔진 구조
    
* JSON 스키마
    
* 상태 관리 설계
    
* 기술 스택 추천 (React / Svelte / Canvas / WebGL 등)
