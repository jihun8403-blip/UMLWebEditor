# UML Web Editor

## Test Strategy – MVP

* * *

## 1. 문서 개요

본 문서는 UML Web Editor MVP의 품질 확보를 위한 테스트 전략을 정의한다.  
테스트의 목적은 기능적 정확성, 데이터 무결성, 사용성, 성능, 안정성을 검증하는 것이다.

본 제품은 브라우저 기반 정적 웹앱이며, UML 요소를 데이터 모델로 관리하는 캔버스 편집기이므로 다음 3가지를 동시에 검증해야 한다.

* 사용자 화면 동작
    
* 내부 상태 및 데이터 일관성
    
* 저장/복구 후 동일성 유지
    

* * *

## 2. 테스트 목표

테스트 목표는 다음과 같다.

* 주요 UML 편집 기능이 PRD 요구사항대로 동작하는지 검증
    
* 요소/관계선/속성 편집 결과가 데이터 모델에 정확히 반영되는지 검증
    
* JSON import/export 시 데이터 손실이나 구조 왜곡이 없는지 검증
    
* Undo/Redo 시 화면과 데이터가 일관되게 복원되는지 검증
    
* 성능 목표를 만족하는지 검증
    
* 브라우저 로컬 환경에서 안정적으로 동작하는지 검증
    

* * *

## 3. 테스트 범위

### 3.1 포함 범위

MVP 테스트 범위는 아래와 같다.

* 단일 캔버스 및 다이어그램 타입 전환
    
* 클래스 / 유스케이스 / 시퀀스 다이어그램 요소 생성
    
* 시퀀스 다이어그램의 Activation 포함 동작
    
* 관계선 생성 및 orthogonal routing
    
* 우측 속성 패널 기반 편집
    
* 요소 선택, 이동, 복제, 삭제
    
* 다중 선택 및 그룹 이동
    
* Undo / Redo
    
* Grid / Snap / 정렬
    
* 검색
    
* JSON 저장 / 불러오기 / import / export
    
* 자동 저장 / 복구
    
* PNG / SVG / PDF export
    
* 성능 및 호환성 검증
    

### 3.2 제외 범위

아래 기능은 MVP 제외 범위이므로 본 테스트 범위에서도 제외한다.

* 장애물 회피형 스마트 라우팅
    
* 코드 생성
    
* 리버스 엔지니어링
    
* XMI import/export
    
* 실시간 협업
    
* Git 연동
    
* 벡터 기반 PDF export

## 4. 테스트 대상 품질 속성

본 프로젝트에서 우선적으로 검증할 품질 속성은 다음과 같다.

### 4.1 기능성

정의된 UML 요소 및 관계선 편집 기능이 정상 동작해야 한다.

### 4.2 정확성

사용자 입력 결과가 화면과 데이터 모델에 동일하게 반영되어야 한다.

### 4.3 무결성

저장, export, import, 복구 이후에도 데이터가 손상되지 않아야 한다.

### 4.4 사용성

주요 기능은 직관적으로 동작해야 하며, 편집 흐름이 자연스러워야 한다.

### 4.5 성능

PRD에 정의된 응답시간 및 렌더링 성능을 충족해야 한다.

### 4.6 안정성

비정상 종료, 잘못된 JSON 입력, 반복 편집 상황에서도 앱이 무너지지 않아야 한다.

## 5. 테스트 유형

### 5.1 단위 테스트

검증 대상:

* 데이터 모델 검증 함수
    
* 관계선 경로 계산 함수
    
* JSON 직렬화 / 역직렬화 함수
    
* 명령(Command) 기반 Undo/Redo 로직
    
* 검색 필터 함수
    

### 5.2 통합 테스트

검증 대상:

* 속성 패널 편집과 캔버스 반영 연계
    
* 관계선 생성과 요소 이동 연계
    
* Undo/Redo와 상태 저장 연계
    
* JSON import 후 렌더링 연계
    

### 5.3 기능 테스트

검증 대상:

* 사용자 기능 시나리오 전반
    
* 요소 생성, 편집, 삭제
    
* 타입 전환
    
* export
    
* 자동 저장 / 복구
    

### 5.4 회귀 테스트

기존 기능 수정 후 아래 영역을 우선 재검증한다.

* 요소 생성/삭제
    
* 관계선
    
* 히스토리
    
* JSON import/export
    
* export 기능
    

### 5.5 성능 테스트

검증 대상:

* 요소 300개 / 관계선 500개 기준 편집 성능
    
* 대용량 JSON import/export 시간
    
* 팬/줌 반응성
    
* 장시간 편집 시 메모리 증가 추이
    

### 5.6 호환성 테스트

검증 대상:

* Chrome
    
* Edge
    
* Firefox

## 7. 테스트 접근 방식

본 프로젝트는 다음 순서로 테스트를 수행한다.

* 다이어그램 타입 전환 시, **비활성 타입의 요소는 숨김 처리되며 데이터는 유지**된다(전환 후 복귀 시 동일 상태로 복원).

### 7.1 1차: 데이터 및 핵심 로직 검증

먼저 단위 테스트로 아래를 고정한다.

* element 생성/수정/삭제 데이터 처리
    
* relationship 데이터 연결
    
* history stack 동작
    
* JSON serialization/deserialization
    

### 7.2 2차: UI-상태 연동 검증

다음으로 통합 테스트에서 아래를 검증한다.

* 우측 패널 편집 결과가 캔버스에 반영되는지
    
* 캔버스 조작이 상태에 반영되는지
    
* Undo/Redo가 UI와 모델에 동시에 적용되는지
    

### 7.3 3차: 사용자 시나리오 검증

기능 테스트에서 실제 편집 흐름을 검증한다.

예:

* 클래스 생성 → 속성 추가 → 관계선 연결 → JSON 저장 → 다시 import
    
* 시퀀스 다이어그램 생성 → lifeline 추가 → activation 추가 → message 연결
    

### 7.4 4차: 비기능 검증

마지막으로 성능, 복구, 브라우저 호환성을 검증한다.

## 8. 결함 심각도 기준

### Critical

앱이 멈추거나 데이터 손실이 발생하는 경우

예:

* 저장 파일이 손상됨
    
* import 후 앱 크래시
    
* undo 후 전체 캔버스 상태 붕괴
    

### High

주요 기능 수행이 불가능한 경우

예:

* 클래스 생성 불가
    
* 관계선 연결 불가
    
* export 실패
    

### Medium

기능은 수행되지만 일부 결과가 부정확한 경우

예:

* 검색 하이라이트 위치 오류
    
* 미니맵 반영 지연
    
* 정렬 결과 오차
    

### Low

사용성 또는 시각적 완성도 문제

예:

* 아이콘 정렬 어긋남
    
* 패널 여백 문제
    
* 경미한 텍스트 겹침
    

* * *

## 9. 합격 기준 (Exit Criteria)

MVP 테스트 완료 기준은 다음과 같다.

* Critical 결함 0건
    
* High 결함 0건
    
* 주요 기능 테스트 케이스 Pass 100%
    
* 전체 테스트 케이스 Pass 95% 이상
    
* 성능 요구사항 충족
    
* JSON import/export 무결성 오류 0건
