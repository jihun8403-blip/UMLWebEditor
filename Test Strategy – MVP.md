# 1. 문서 개요

본 문서는 **UML Web Editor MVP의 품질 확보를 위한 테스트 전략을 정의한다.**

본 시스템은 브라우저에서 실행되는 **캔버스 기반 UML 다이어그램 편집기**이며, 다음 세 가지 계층이 동시에 올바르게 동작해야 한다.

* 사용자 인터페이스 동작
    
* 내부 데이터 모델 상태
    
* 저장 및 복구 데이터 무결성
    

따라서 테스트 전략은 다음 세 가지 일관성을 검증하는 것을 목표로 한다.

Plain textUI 상태  
Canvas 렌더링 상태  
데이터 모델 상태

이 세 상태는 항상 동일해야 한다.

* * *

# 2. 테스트 목표

본 테스트의 주요 목표는 다음과 같다.

1. UML 편집 기능이 PRD 요구사항대로 동작하는지 검증
    
2. 사용자 편집 결과가 데이터 모델에 정확히 반영되는지 확인
    
3. JSON 저장 및 복구 시 데이터 무결성 유지 여부 검증
    
4. Undo / Redo 동작의 일관성 검증
    
5. 캔버스 기반 상호작용의 안정성 검증
    
6. 성능 목표 충족 여부 확인
    
7. 브라우저 환경에서 안정적인 동작 확인
    

* * *

# 4. 테스트 대상 품질 속성

본 프로젝트에서 검증할 주요 품질 속성은 다음과 같다.

## 4.1 기능성

정의된 UML 요소 및 편집 기능이 정상적으로 동작해야 한다.

## 4.2 정확성

사용자 편집 결과가 화면과 데이터 모델에 동일하게 반영되어야 한다.

## 4.3 데이터 무결성

저장 / export / import / 복구 이후에도 데이터가 손상되지 않아야 한다.

## 4.4 사용성

주요 편집 기능은 직관적으로 동작해야 한다.

## 4.5 성능

편집 반응성과 렌더링 성능이 목표 기준을 만족해야 한다.

## 4.6 안정성

장시간 사용 및 예외 상황에서도 앱이 정상 동작해야 한다.

# 8. 결함 심각도 기준

## Critical

시스템 동작 불가 또는 데이터 손실

예

* Import 후 앱 크래시
    
* 저장 데이터 손상
    
* Undo 후 전체 상태 붕괴
    

* * *

## High

주요 기능 수행 불가

예

* 요소 생성 실패
    
* 관계선 생성 불가
    
* Export 실패
    

* * *

## Medium

기능은 동작하지만 결과가 일부 부정확

예

* 검색 하이라이트 오류
    
* 정렬 오차
    
* 렌더링 위치 미세 오류
    

* * *

## Low

UI 또는 시각적 문제

예

* 아이콘 정렬
    
* 텍스트 겹침
    
* 패널 여백
    

* * *
